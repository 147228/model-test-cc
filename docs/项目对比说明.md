# 项目对比说明

## 新工具 vs 原有项目

本工具整合了 `cc测评提示词` 和 `gemini3 画图` 两个项目的优点，并做了重大改进。

## 功能对比

| 功能特性 | cc测评提示词 | gemini3画图 | **新工具** |
|---------|------------|-----------|---------|
| **文生文测评** | ✅ | ❌ | ✅ |
| **文生图测评** | ❌ | ✅ | ✅ |
| **图形界面** | ❌ | ❌ | ✅ |
| **配置方式** | 代码修改 | 代码修改 | GUI配置 |
| **测试用例** | 25个 | 70个 | 20个（精选） |
| **并发线程** | 20线程 | 30线程 | 1-30可配置 |
| **网站生成** | ✅ | ✅ | ✅ |
| **启动方式** | 命令行 | 命令行 | 双击启动 |
| **模型切换** | 改代码 | 改配置 | 界面输入 |
| **进度显示** | 命令行 | 命令行 | GUI进度条 |
| **日志查看** | 文件 | 文件 | 实时GUI |
| **结果预览** | 浏览器 | 浏览器 | 一键打开 |

## 使用体验对比

### 原有流程（cc测评提示词）

```bash
1. 用编辑器打开 batch_test.py
2. 找到 API_KEY 行，修改密钥
3. 找到 BASE_URL 行，修改地址
4. 找到 MODEL 行，修改模型
5. 保存文件
6. 打开命令行
7. 输入: python batch_test.py
8. 等待完成（看不到实时进度）
9. 手动运行: python generate_ui.py
10. 浏览器打开 index.html
```

**问题：**
- 需要会编程
- 修改代码容易出错
- 命令行操作门槛高
- 进度不可见

### 原有流程（gemini3 画图）

```bash
1. 复制 config.example.json 为 config.json
2. 用编辑器打开 config.json
3. 修改 API Key
4. 修改模型名称
5. 保存文件
6. 打开命令行
7. 运行: venv\Scripts\python.exe tools\batch_generate_fast_clean.py 1 70
8. 运行: venv\Scripts\python.exe tools\extract_images_from_json.py
9. 运行: venv\Scripts\python.exe tools\generate_website_enhanced.py
10. 浏览器打开 output\website\index.html
```

**问题：**
- 步骤繁琐
- 需要记住多个命令
- 虚拟环境路径复杂
- 容易遗漏步骤

### 新工具流程

```bash
1. 双击 启动.bat
2. 在界面上填写 API URL
3. 填写 API Key
4. 填写模型名称
5. 点击"保存配置"
6. 选择测评类型
7. 点击"开始测评"
8. 完成后点击"打开输出目录"
9. 双击 index.html
```

**优势：**
- ✅ 无需编程知识
- ✅ 可视化操作
- ✅ 实时进度显示
- ✅ 一键完成所有流程
- ✅ 不会操作错误

## 技术改进

### 1. 架构设计

| 方面 | 原项目 | 新工具 |
|------|--------|--------|
| 代码组织 | 单文件大脚本 | 模块化设计 |
| 配置管理 | 硬编码 | JSON持久化 |
| 错误处理 | 基础 | 完善的异常捕获 |
| 日志系统 | 文件日志 | 实时GUI日志 |
| 用户反馈 | 无 | 进度条+状态提示 |

### 2. 测试用例优化

**原项目的问题：**
- cc测评提示词: 25个案例，有些过于复杂（如Mac OS模拟器11453 tokens）
- gemini3画图: 70个提示词，数量多但质量参差不齐

**新工具的改进：**
- 精选20个案例（10文生文 + 10文生图）
- 难度梯度合理（简单→中等→高难）
- 每个案例都有明确的测试目标
- 提示词经过优化，效果更好

### 3. 网站展示

| 特性 | cc测评 | gemini3画图 | 新工具 |
|------|--------|-----------|--------|
| 设计风格 | 粉色可爱 | 深色现代 | 渐变专业 |
| 响应式 | ✅ | ✅ | ✅ |
| 图片预览 | ❌ | ✅ | ✅ |
| HTML演示 | ✅ | ❌ | ✅ |
| 搜索功能 | ❌ | ✅ | ❌（计划v2.0） |
| 统计面板 | ✅ | ✅ | ✅ |

### 4. 性能优化

```
并发测试性能对比（20个案例）：

原项目（cc测评）:
- 固定20线程
- 耗时: 约4分钟
- 不可调节

原项目（gemini3画图）:
- 固定30线程
- 耗时: 76秒（70个案例）
- 代码修改才能调节

新工具:
- 1-30线程可配置
- 10线程: 约15分钟
- 20线程: 约8分钟
- 30线程: 约5分钟
- GUI滑块实时调节
```

```

```
